# Mode: Guided

For a user building identity from scratch, or doing it deliberately for the first time.
The job is to walk the five components in order, **explaining the why at each step**, so
the user finishes understanding *their own* brand — not just holding a palette someone
handed them. Move at their pace; one component at a time; confirm before advancing.

Load `references/core-principles.md` (the frameworks) and `references/components.md` (the
per-component method) alongside this file.

## Posture

This is a teaching conversation, not a generator. The user is likely non-designer; the
value is that they leave able to defend every choice. Explain each framework briefly the
first time it comes up, in plain language, and tie every recommendation back to *their*
personality and onliness line. When you offer options, give a recommendation with a reason —
not a menu that pushes the decision back onto them.

## The flow

**Step 0 — ground the strategy.** Check for positioning/personas. If absent, ask 2–3
questions to surface the brand's drive and differentiation before touching anything visual.
Don't block on it, but make the gap visible (a brand with nothing to express will drift to
defaults).

**Step 1 — Personality.** Land the archetype, the three constraining adjectives, the
onliness line, and the north-star sentence. Do not move on until the three adjectives each
exclude something — this is the load-bearing step. Everything downstream is derived from it.

**Step 2 — Color.** Translate the adjectives into chromatic direction. Decide fit-vs-break
the category. Build the role set (dominant, accent, base, semantic). Name by role. Verify
contrast. Explain the color-is-contextual caveat so they don't over-trust pop "color
psychology."

**Step 3 — Typography.** Choose the voice of the type from the adjectives, pick two
families, define the signature move, set the hierarchy, and check the smallest real size.

**Step 4 — Logo.** Start from the wordmark + one detail. Run the survival tests (16px, one
color, light/dark). Deliver a direction with rationale.

**Step 5 — Documentation.** Assemble it all into a one-page brand doc + tokens, and offer
to write the files into their project. Name a few "rules with names." This is where the
session becomes reusable.

## Checkpoints

After each component, reflect it back in one or two sentences and ask for a thumbs-up
before continuing — e.g. "So the personality is an Explorer brand that's tough, free, and
plainspoken; everything visual should reward discovery and feel built-for-use, not
showroom-pretty. Good to build color on that?" This keeps the user steering and catches a
wrong personality before it propagates into color and type.

## Close

End with the full **Output always includes** set from SKILL.md (personality statement,
color system, type system, logo direction, brand doc + tokens, what-it-rules-out, open
questions). Then hand off explicitly: the verbal half of this same identity is
`/tone-of-voice`, and the words that sell it are `/product-copy`. The visual identity you
just built is what those two will sound like out loud.
