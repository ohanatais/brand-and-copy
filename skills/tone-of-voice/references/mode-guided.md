# Mode: Guided

For a user defining voice and tone for the first time. Walk the five layers in order,
explaining the why, so the user leaves able to write on voice themselves — not just holding
a guide. One layer at a time; confirm before advancing.

Load `references/core-principles.md` and `references/components.md` alongside this file.

## Posture

Teaching conversation, not a generator. The user is likely non-designer and non-copywriter;
the value is that they can hit the voice consistently afterward. Explain voice-vs-tone and
the four dimensions plainly the first time they come up. Anchor every recommendation in
*their* brand personality. Use the user's real product strings as raw material wherever
possible — abstract examples teach less than their own copy rewritten.

## The flow

**Step 0 — ground the personality.** Pull brand identity / positioning. If absent, derive
three working personality adjectives before touching voice. Make the gap visible without
blocking.

**Step 1 — Voice.** Land the three "how it writes" adjectives (each must exclude something)
and the default setting on NN/g's four dimensions. Write the one-paragraph "who's talking."
This is load-bearing — don't advance until the adjectives are sharp.

**Step 2 — Tone.** List the contexts that actually recur in their product. For each, define
how the voice shifts — especially across emotional states (win, error, empty, confused).
One-line rule per context.

**Step 3 — Lexicon.** Build words-to-use and a named banned list, with the on-voice swap for
each banned item. Make the banned list *theirs*, not a generic anti-AI list.

**Step 4 — Microcopy.** Rewrite their real error messages, empty states, onboarding opener,
and key buttons on voice. This is where the abstract becomes obviously useful.

**Step 5 — Guardrail & doc.** Assemble the voice doc and offer to write it to a file. If
they'll generate copy at scale or with AI, offer the lightweight consistency check.

## The "say it three ways" anchor

Early in the session, do one "say it three ways" on a real product message (on voice / off
voice / on the edge). It's the fastest way to make the voice legible and to confirm you and
the user agree on what "right" sounds like before building the rest. Produce 3–4 total by
the end.

## Checkpoints

After each layer, reflect it back in a sentence and get a thumbs-up. Catching a wrong voice
setting at Step 1 saves rewriting every microcopy example later.

## Close

Deliver the full **Output always includes** set from SKILL.md, write the voice doc, and hand
off to `/product-copy` — which inherits this voice to write the high-stakes conversion copy.
Note that the visual twin of this voice is `/brand-identity`; if that isn't done, flag that
the product's look and words should be defined together.
