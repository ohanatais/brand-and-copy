# Mode: Asset

For a user who knows their message and voice and wants a specific asset **produced
efficiently** — a landing page, an onboarding flow, a launch email — not a teaching walk.
Decisive output, then refine. Still derived from the reader, never a swipe file.

Load `references/core-principles.md` and `references/components.md` alongside this file.

## Posture

Respect the upstream work. Pull positioning, voice, and any real customer language, and treat
them as source. Make the awareness call, then draft the whole asset in the brand voice in one
pass, labeled so the user can see what each part does. The deliverable is copy ready to ship
and test.

## The flow (one pass, then refine)

1. **Call the reader & stage** in one line — it justifies the structure. If the asset will be
   hit by readers at different stages (e.g. an evergreen homepage), say which stage you're
   optimizing for and why.
2. **State the value proposition** you're building on (reuse positioning's if it exists).
3. **Draft the asset** against its anatomy (`components.md`), dropping parts this reader's
   stage doesn't need. Write in the defined voice. Apply specificity, the "so what" ladder,
   and voice-of-customer language as you go — not as a later edit.
4. **Handle objections and proof** inline; match proof to each claim.
5. **Write the CTA(s)** — specific, low-friction, one primary. Include **2–3 headline and CTA
   variations** to test.

Present the asset as a coherent whole with light labels (what each section/line is doing), so
the user reacts to the copy *and* can learn the structure.

## Where to slow down

Even fast, don't skip:
- The **awareness call** — wrong stage means wrong opening, and everything downstream
  inherits the error.
- The **5-second test** on the hero.
- **Real customer language** where it exists — it beats anything invented.
- **Voice match** — confirm the draft sounds like the defined voice, not generic persuasion.

## Close

Deliver the full output set, write the copy to a file if useful, and hand off to ship/test or
`/go-to-market`. Flag what's a hypothesis and what you'd most want to validate with real
readers first (usually the hero headline and the primary CTA).
