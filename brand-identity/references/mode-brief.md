# Mode: Brief

For a user who already has strategy — positioning, personas, or at least a clear onliness
line — and wants the **brand brief and tokens produced efficiently**, not a teaching walk.
Less explanation, more decisive output. Still derived, never defaulted.

Load `references/core-principles.md` and `references/components.md` alongside this file.

## Posture

The user has done the upstream thinking; respect it. Pull their positioning/personas/
onliness line and treat it as the source of truth for the personality. Make confident,
reasoned recommendations across all five components in one pass, then refine where they
push back. The deliverable is the brand doc + tokens, ready to drop into their project.

## The flow (one pass, then refine)

1. **Restate the personality** from their strategy: primary archetype, three constraining
   adjectives, onliness line, north-star sentence. If their inputs don't yet imply a crisp
   archetype or adjectives, derive a proposal and flag it for confirmation — this is the one
   place worth a beat of friction, because everything else hangs on it.
2. **Propose the color system** in full: dominant, accent, bases, semantic — each named by
   role, with intended read and contrast notes. State the fit-vs-break decision and why.
3. **Propose the type system**: families + pairing rationale, the signature move, hierarchy.
4. **Propose the logo direction**: wordmark + one detail, survival notes.
5. **Write the brand doc + tokens.** Offer to write them to files matching the user's stack
   (CSS custom properties, a Tailwind/shadcn theme, or a JSON token set). Include the
   do/don't list and the named rules.

Present 2–4 as a single coherent proposal so the user reacts to a *whole identity*, not a
sequence of isolated choices — coherence across components is the thing being judged.

## Where to slow down

Even in fast mode, do not skip:
- **Adjectives that exclude something** — if their strategy yields generic adjectives,
  sharpen them before proceeding.
- **Contrast verification** on every text/surface pair.
- **The smallest-size check** for type and logo.

These are cheap and catch the failures that are expensive to fix after the brand ships.

## Close

Deliver the full **Output always includes** set, write the files, and hand off to
`/tone-of-voice` then `/product-copy`. Note anything still held as hypothesis so the user
knows what to revisit once real users react to the brand.
